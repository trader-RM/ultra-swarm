# v6.79.0

## Overview

This release integrates the v6.69.0–v6.79.0 upstream feature set into the Ultra Swarm fork, preserving all 45 ECC specialist agent registrations and delegation permissions. The three-way merge brings critic hallucination verification, curator knowledge agents, scope persistence, parallel dispatch, and an expanded tool surface.

**No breaking changes.** All existing configurations continue to work.

## New Features

### Critic Hallucination Verifier (opt-in)

A new QA gate for verifying API signatures, factual claims, citations, and code references at PHASE-WRAP. Not enabled by default — activate via the `hallucination_guard` QA gate:

```json
{
  "qa_gates": {
    "hallucination_guard": true
  }
}
```

When enabled:
- Architect delegates to `@critic_hallucination_verifier` before phase completion
- Verdict persisted via `write_hallucination_evidence` tool
- `phase_complete` blocked unless `.swarm/evidence/{phase}/hallucination-guard.json` exists with APPROVED verdict

### Curator Knowledge Agents

Two new support agents for knowledge consolidation at session boundaries:

| Agent | Trigger | Purpose |
|-------|---------|---------|
| `curator_init` | Session start | Consolidates prior session knowledge; flags contradictions |
| `curator_phase` | Phase complete | Extends digest with phase outcomes; observes deviations |

Curator agents are dispatched via `MODE: EXPLORER` with observational language only (no directives).

### Critic Oversight

An autonomous oversight variant of the critic for council and quality gate enforcement. Used when `critic_oversight` is enabled in the QA gate profile.

### Declare Scope Persistence

`declare_scope` now persists the coder's allowed write set to disk (`.swarm/scopes/scope-${taskId}.json`), surviving cross-process delegation. Without a `declare_scope` call, everycoder Edit/Write is denied.

Architect **must** call `declare_scope({ taskId, files })` before every coder delegation — this is enforced as Rule 1a in the architect prompt.
